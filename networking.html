Networking Basics Cheat Sheet

OSI Model

Layer Name Purpose Example
7 Application User-facing apps (HTTP, FTP, email). Web browsers, Gmail, APIs.
6 Presentation Data formatting (encryption, compression). SSL/TLS, JPEG/MPEG encoding.
5 Session Manages connections between devices. VPNs, NetBIOS.
4 Transport End-to-end communication (TCP/UDP). Port numbers, flow control.
3 Network Routing data between networks (IP addresses). Routers, IP packets.
2 Data Link Direct communication between devices (MAC). Switches, Ethernet frames.
1 Physical Physical hardware (cables, Wi-Fi signals). Fiber optics, hubs, NICs.

How to Memorize the OSI Model

Mnemonic: "All People Seem To Need Data Processing"

  • Application → All
  • Presentation → People
  • Session → Seem
  • Transport → To
  • Network → Need
  • Data Link → Data
  • Physical → Processing

TCP/IP Model

Layer Name Purpose Example
4 Application User-facing apps (HTTP, FTP, DNS). Web browsers, email clients.
3 Transport End-to-end communication (TCP/UDP). Port numbers, flow control.
2 Internet Routing data between networks (IP addresses). Routers, IP packets.
1 Network Access Physical hardware and MAC addresses. Ethernet, Wi-Fi.

Subnetting Basics

Subnet Mask: Defines network vs. host portions of an IP address.

  • /24: 255.255.255.0 (256 IPs, 254 usable).
  • /16: 255.255.0.0 (65,536 IPs).

Common Networking Tools

Tool Purpose Example
Wireshark Packet analysis (Layers 2–7). Detect malicious traffic.
Nmap Network scanning (Layers 3–4). Find open ports on a server.
Ping Test connectivity (Layer 3). Check if a device is online.
Traceroute Trace packet path (Layer 3). Identify network hops.
